Change in Top Performing Steam Genres and Top Publishers Analysis

The final challenge for Google Data Analytics Professional's course was to finish a project to showcase what I've learned called capstone project. I was presented with three options: Analysis of a bike-sharing company, a wellness technology company or choosing my own project. I decided to go with the later option.

Initially I was considering doing an analysis on a marketing topic but after coming across some good datasets about steam's player count throughout the years I couldn't resist picking it as my topic. After having picked I topic I decided to start with the method I learned in Google's course, the six data analysis phases.


Ask

The first phase is to determine the goal of the project and define it's scope. This is where I get to talk to the stakeholders to understand their requirements. I defined my stakeholders as a game publisher interested in learning more about changes in trending genres throughout the years and product effectiveness of publishers. To limit my scope and have a tangible goal and defined three questions that I had to answer by the end of this analysis.

  • How has the most popular genres changed over the past years?
  • What were the most popular genres by monthly player count in 2021?
  • Which developers produce the best games in genres answered by the previous question using Metacritic scores as a metric?

Prepare

This is where I locate where data relevant to this analysis is located and it's integrity. I have to make it's credible and free of security and privacy concerns.

I found two great 3rd party datasets in Kaggle for Steam historic stats from Steam Charts and Metacritic scores.

Steam is a vide game distribution service that most game offered and played on. It provides stats on daily and monthly average users for each game but doesn't offer historic data. Steam Charts is a website that's been operating since 2012 that accesses Steam's stats using their API and have stored all the data since 2012.

Metacritic is a website that aggregates reviews of video games and has two sets of score each game, meta score based on critic reviews and user score based reviews of the general public.

One issue that might skew the result is that some lesser known games might have different names in the two datasets causing them to be left out. Also Steam appears to provide different stats for different DLCs of a game so the total number of average players might be less in some scenarios.

After confirming that the data is upto date, reliable and comprehensive I decided to proceed to the next phase


Process

This is where I familiarize myself with the data and employ data processing techniques such as data type conversion, feature creation, and missing value imputation. But first I familiarized myself with the two datasets.

Metacritic columns:

  • Game name: Name of the video game this row is referring to.
  • Meta score: Score based on critics reviews.
  • User score: Score based on user reviews.
  • Platform: Platforms the game is available on.
  • Description: Description of the game.
  • Url: Link to the original Metacritic website.
  • Developer: Name of the developer of the video game.
  • Genre: A list of genres used to define the game.
  • Type: Whether the game is singleplayer of multiplayer.
  • Rating: Audience rating of the game e.g. E, M, T

SteamCharts columns:

  • Name: Name of the video game this row is referring to.
  • Month: Month and year of the recorded row stats.
  • Avg. players: Average players for the registered month.
  • Gain: Change in average player count since last month.
  • %Gain: Percentage change in average player count since last month.
  • Peak players: Highest number of players in any given time since the release of the game.

Then I proceeded with data processing. The following is a list of techniques I applied to the datasets

  • Updated columns names to follow naming conventions
  • Removed rows in Steam data with "Last 30 days" in date
  • Converted columns data, average_players, perc_player_change, and player_change to appropriate data type
  • Added average score column to Metacritic as part an average of the two scores
  • Converted genre from string to list then split it into multiple rows for each genre
  • Dropped remaining rows with na values
  • Dropped extra columns that are unnecessary for this analysis such as platform, description, url, type, rating, and etc.

This phase took the longest time as I tried to make sure the data was fully clean and ready for the next phase.


Analyze and Share

I started my analysis using R by merging the two datasets on game name. This resulted in a table with 486k rows, 2168 games, 139 genres and 1329 developers. When I prepared the data I accepted the limitations of the two datasets and the fact that many games will be deleted from the list.

After finished the analysis I decided to showcase the result using Tableau.
