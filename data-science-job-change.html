Job Change of Data Scientists

For this project I decided to acquire data from kaggle dataset HR Analytics: Job Change of Data Scientists. This project goes through basic analysis of the data, pre-processing and applying a Random Forest model to try to predict whether a person is likely to stay in a company or leave it.

The entire code as well as more detailed result can be found in github.

I used Google Colab to host the code and data and run the code for simplicity and ease of use.

Analysis result
  • The data contains a lot of null values which needs to be addressed.
  • The data is in label form and will have to be converted to numerical values to input into models for better accuracy.
  • The data is imbalanced especiialy in variables like target and gender. While there are also imbalances in variables like relevent experience, university enrollment, education level, major discipline and company type these imbalances are to be expected as they are usually the requirements for data science positions.
  • The correlation table provides valuable insight between the valuable like how higher city development index directly correlates to higher experience.
Pre-processing
  • Train and test data were combined for consistency throughout the process.
  • First all the values have to be encoded to numerical values. LabeleEncoder was used on nominal values and direct mapping was applied on oridnal values to maintain the order.
  • To handle the missing information there many option like remove rows/columns with missing data, applying average, training a regressor to predict the values. KNN Imputer was used on the data to avoid losing any data and the simplicty of the approach. The values were also rounded because they are categorical.
  • Train and test were seperated in preparation for model training.
Model Training
  • Random Forest was chosen as the predictor model.
  • After training the model achieved 99% on training, 77% on validation and 75% on testing.
  • The model was clearly overfitting so over sampling with SMOTE was used to balance the data more. The validation accuracy increased to 84% and testing reduced to 76%.
  • Another layer of oversampling was applied to increase the dataset size more which is resulted in increase in validation accuracy but again reduction in testing.
Potential improvements
  • Testing with different ways to fill the missing values.
  • Hyoeroarameter tuning the Random Forest model.
  • Applying a different model like boosted tree model.
